# /storybook - Component Stories

$ARGUMENTS

---

## Task

Create interactive component documentation with stories.

### Steps:

1. **Identify** - List component variants and states
2. **Build** - Create stories for each variant
3. **Document** - Add usage docs and prop tables
4. **Test** - Visual regression tests for each story

---

## Usage Examples

```
/storybook Button         # Create Button stories
/storybook --all          # Generate stories for all components
```
