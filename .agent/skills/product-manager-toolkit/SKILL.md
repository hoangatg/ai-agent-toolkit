# Product Manager Toolkit

> Build the right thing, not just build things right.

---

## 1. Product Discovery

### Framework Selection

| Framework | Best For |
|-----------|----------|
| **Jobs to Be Done** | Understanding true user needs |
| **Design Thinking** | Innovative solutions |
| **Lean Startup** | Validating assumptions fast |
| **User Story Mapping** | Feature prioritization |

### Discovery Questions

| Question | Purpose |
|----------|---------|
| Who is the user? | Target audience clarity |
| What problem are we solving? | Problem validation |
| How do they solve it today? | Competitive landscape |
| What does success look like? | Measurable outcomes |
| What's the smallest useful version? | MVP scoping |

---

## 2. Prioritization Frameworks

### RICE Scoring

| Factor | Calculation |
|--------|------------|
| **Reach** | How many users affected per quarter |
| **Impact** | Massive (3x), High (2x), Medium (1x), Low (0.5x), Minimal (0.25x) |
| **Confidence** | High (100%), Medium (80%), Low (50%) |
| **Effort** | Person-months |
| **Score** | (Reach × Impact × Confidence) / Effort |

### MoSCoW

| Category | Definition |
|----------|-----------|
| **Must have** | Launch blocker |
| **Should have** | Important, not critical |
| **Could have** | Nice to have |
| **Won't have** | Out of scope (this release) |

## 3. PRD Structure

| Section | Content |
|---------|---------|
| **Problem** | What pain point, for whom |
| **Hypothesis** | What we believe will happen |
| **Success metrics** | How we'll measure |
| **User stories** | As a..., I want..., So that... |
| **Requirements** | Functional + non-functional |
| **Design** | Wireframes, flows |
| **Technical notes** | Constraints, dependencies |
| **Timeline** | Milestones and deadlines |

---

## 4. User Stories

### Format

```
As a [persona]
I want to [action]
So that [benefit]
```

### Acceptance Criteria (Given-When-Then)

```
Given [context]
When [action]
Then [expected result]
```

## 5. Metrics & KPIs

### North Star Metrics

| Business Type | North Star |
|--------------|-----------|
| **SaaS** | Weekly Active Users (WAU) |
| **E-commerce** | Revenue per session |
| **Marketplace** | Transactions per month |
| **Content** | Time spent reading |

### AARRR Funnel

| Stage | Metric |
|-------|--------|
| **Acquisition** | Sign-ups, traffic sources |
| **Activation** | Completed onboarding, "aha moment" |
| **Retention** | D1/D7/D30 retention rates |
| **Revenue** | ARPU, LTV, conversion rate |
| **Referral** | NPS, viral coefficient |

---

## 6. Anti-Patterns

| ❌ Don't | ✅ Do |
|----------|-------|
| Build everything users ask for | Understand the problem behind requests |
| Ship without metrics | Define success criteria before building |
| Feature factory | Outcome-driven development |
| Skip user research | Talk to users weekly |
| Waterfall PRDs | Iterate and learn |

---

> **Remember:** The best product managers say "no" more than "yes." Every feature has a maintenance cost — prioritize ruthlessly.
